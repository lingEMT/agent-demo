# HelloAgents 智能旅行规划助手 - 项目规范

## 更新记录规范

每次新增功能或完成重要修改时，必须在 `update/` 目录下创建更新记录。

### 规范要求

1. **创建子文件夹**：在 `update/` 下以功能名称创建子文件夹，使用小写字母+连字符命名
   - 示例：`update/redis-cache/`

2. **生成 Markdown 记录文件**：在子文件夹中创建 `.md` 文件，文件名使用大写+下划线风格
   - 示例：`update/redis-cache/REDIS_CACHE_IMPLEMENTATION.md`

3. **记录内容应包含**：
   - **概述**：功能/修改的简要说明
   - **新增文件**：列出所有新增的文件及说明
   - **修改文件**：列出所有修改的文件及变更要点
   - **依赖变更**：新增或移除的依赖包
   - **配置说明**：新增的配置项及示例

### 示例结构

```
update/
├── redis-cache/
│   └── REDIS_CACHE_IMPLEMENTATION.md
└── <新功能名称>/
    └── <功能描述>.md
```

### 注意事项

- 不要删除或修改已有的更新记录
- 每个功能独立一个子文件夹，不要混放
- 记录内容要清晰简洁，重点突出变更内容
